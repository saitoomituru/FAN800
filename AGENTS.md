# AGENTS.md (Root) — FAN800

このファイルはAIエージェント（Claude等）がこのプロジェクトで作業する際の前提・方針を記述する。
**サブディレクトリにも `AGENTS.md` が存在する場合、階層が深いものほど優先度が高い**。

---

## 用語定義（ハルシネーション防止）

AIエージェントはここに記載された正式定義のみを使うこと。

| 識別子 | 正式名称 | 意味・補足 |
|-------|---------|-----------|
| **FAN800** | Fannel Ad-hoc Network 800 | ESP32 BLEメッシュで構成されるサイコミュファンネル群。"Fannel" はファンネル（funnel）の造語表記。 |
| **OND800** | OBS NDI Dominator 800 | モビルスーツ本体。FAN800の上位レイヤー。BLE GATTでイベント言語を送ってくる相手。 |
| **SAO800** | Service Add-on OBS 800 | OBSプラグイン。FAN800は直接関与しない。 |
| **イベント言語** | — | `FIRE_SLIME_MEDIUM` のような意図ベースのコマンド。ハードウェアプリミティブ（GPIO_PIN等）は禁止。 |
| **自己申告パケット** | — | FAN800がOND800に接続時に送る機能・制約宣言JSON。 |
| **mutex_group** | — | 競合する物理演出グループ。`pyro`と`wet`は同時禁止。 |

> **ハルシネーション防止注記**
> - FAN800 の "FAN" は扇風機・ファンではない。**Fannel（ファンネル造語）+ Ad-hoc Network** の略。
> - FAN800はOND800の「子機」ではあるが、OND800が落ちても自律動作する独立ユニット。
> - イベント言語は「意図」を表現する。物理制御プリミティブを直接指定するコマンドは設計違反。

## このプロジェクトの性質

- **コアビジョン**: 演者がソロでプロ級の演出環境を制御できるファンネルプラットフォーム。
- ESP32-S3ファームウェアが中心。全機種で共通のBLE GATTプロトコルを実装する。
- 機種バリエーション（AC/PWM/IR/MD/DMX/MD-DMX）はアクチュエータが違うだけで通信仕様は同一。
- 開発リソースは投げ銭とジャンク箱。停滞・放置があっても「プロジェクト停止」と判断しない。
  再開時は `/notes` を読んで状況を再把握すること。

---

## アーキテクチャ原則（違反禁止）

1. **イベント言語のみを受け付ける** — `GPIO_PIN_4_HIGH_500ms` のようなハードウェアプリミティブは拒否する設計にすること。
2. **自己申告パケットを必ず実装する** — OND800が接続した瞬間に capabilities を送信しないと OND800 はデバイスを認識できない。
3. **物理安全チェックはFAN800の責務** — OND800を盲信しない。温度・クールダウン・過電流は自律リジェクト。
4. **OND800なしで自律動作できる** — MIDIルール受信後は内部RTCでシーケンスを実行。通信断でも継続。
5. **上位レイヤーに依存しない** — FAN800はOND800・SAO800・OBSの内部状態を知らなくていい。

## 実行環境別の役割

### クラウド側（claude.ai 等）
- ドキュメント・AGENTS.md整備
- ファームウェア設計・プロトコル設計
- `/notes` への設計議論の記録

### 実機側（ESP32 + USB接続）
- Arduino IDE / PlatformIO でビルド・書き込み
- BLE GATT通信の動作確認
- 実機特有の問題記録 → `/notes` に追記

## エージェント間連携・引き継ぎ

- 作業記録は **`/notes/YYYY-MM-DD_短い説明.md`** に残す。
- GitHub IssuesはAIエージェントからアクセスできない場合があるため、`/notes` を正とする。
